---
name: No confirmation before making changes
description: User wants changes made directly without asking for confirmation first
type: feedback
---

Just make the change — don't ask "want me to do X?" before doing it. Present the plan briefly if needed, then execute immediately.

**Why:** User finds it annoying to be asked for confirmation on every small change.

**How to apply:** When the approach is clear, implement it directly. Only pause to confirm for risky/destructive actions (deleting files, breaking changes, etc.).
